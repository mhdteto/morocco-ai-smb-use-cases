# How to Use This Library

## Overview

This repository is a practical AI use case library for Moroccan SMBs.

It helps business owners, consultants and operators answer one simple question:

```txt
Where can AI create practical value in this business?
```

The goal is not to adopt AI everywhere.

The goal is to identify focused use cases that are:

- realistic;
- affordable;
- measurable;
- easy to test;
- useful for the business;
- aligned with existing workflows.

---

## Recommended usage

Use this library in five steps:

1. Choose the sector closest to the business.
2. Review the relevant use cases.
3. Score each use case by complexity and value.
4. Select one first pilot.
5. Measure the result before scaling.

---

## Step 1 — Identify the business sector

Start with the closest sector:

- Real estate.
- Retail.
- Education.
- Hospitality.
- Professional services.
- B2B sales.
- Local services.
- E-commerce.

If the business does not fit perfectly, choose the sector with the closest workflow.

---

## Step 2 — Identify repetitive work

AI and automation usually create value where work is repetitive.

Look for tasks such as:

- answering similar questions;
- qualifying leads;
- preparing sales messages;
- generating reports;
- creating content briefs;
- summarizing calls;
- classifying customer requests;
- updating spreadsheets;
- routing inquiries;
- analyzing reviews.

---

## Step 3 — Prioritize use cases

Score each use case using four dimensions:

| Dimension | Question |
|---|---|
| Business value | Does it save time, improve revenue, reduce errors or improve customer experience? |
| Complexity | Is it easy to implement with current tools and data? |
| Data readiness | Is the required information available and clean enough? |
| Adoption readiness | Will the team actually use it? |

Start with use cases that have high value and low or medium complexity.

---

## Step 4 — Start with a pilot

A good AI pilot should be:

- narrow;
- measurable;
- low risk;
- easy to explain;
- connected to a real workflow;
- reviewed by humans;
- limited to a short test period.

Examples:

- AI-assisted lead qualification.
- WhatsApp FAQ assistant.
- Sales follow-up draft generator.
- Review analysis summary.
- Meeting summary to CRM notes.
- Content brief generator.

---

## Step 5 — Measure before scaling

Measure:

- time saved;
- quality improvement;
- response time;
- team adoption;
- customer experience;
- number of manual steps reduced;
- error reduction;
- revenue or pipeline impact if measurable.

Do not scale a use case only because it looks impressive.

Scale it when it proves value.
