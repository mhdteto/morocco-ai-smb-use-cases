# Risks and Limitations

## Overview

AI can create practical value for SMBs, but it also creates risks when used without clear process and human review.

---

## 1. Inaccurate output

AI may generate inaccurate, outdated or incomplete information.

Controls:

- Require human review.
- Ask AI to separate facts from assumptions.
- Verify important information.
- Use trusted source documents.
- Avoid using AI output directly for critical decisions.

---

## 2. Generic content

AI-generated content may sound generic.

Controls:

- Add business context.
- Provide real customer examples.
- Use brand tone guidelines.
- Edit manually before publishing.
- Use prompts with clear constraints.

---

## 3. Confidentiality risk

AI tools may receive sensitive information if users paste private data.

Controls:

- Do not paste passwords, API keys or confidential client data.
- Use approved tools.
- Redact sensitive fields.
- Define internal usage rules.
- Train team members.

---

## 4. Poor workflow fit

An AI use case may fail if it does not match how the team works.

Controls:

- Start with existing workflows.
- Map the process before automating.
- Test with real users.
- Keep the first pilot small.
- Collect feedback.

---

## 5. Data quality issues

Automation depends on input quality.

Controls:

- Clean data before automation.
- Standardize fields.
- Use required fields.
- Validate inputs.
- Add fallback handling.

---

## 6. Over-automation

Some businesses try to automate too much too early.

Controls:

- Start with assistive workflows.
- Keep human approval.
- Avoid fully autonomous customer-facing actions at the beginning.
- Measure before scaling.

---

## Practical rule

Start with AI assistance, not full autonomy.

The best first projects usually help humans work faster and more consistently.
